# Non-official source audit & migration plan

_Generated 2026-06-20 by a 6-finder multi-agent audit workflow (assets · data-JSON · hardcoded tables · network · extractor-inputs · comments/docs) → adversarial verify+plan per finding. 25 raw findings → 16 deduplicated tickets; 0 false positives._

**Goal:** every embedded data/asset input should come from an **official** source — the Ankama CDN or the local game client (decoded `bdata` tables / `gui.jar`). This file tracks what is **not** there yet and exactly how to fix each. Every ticket below is self-contained: a fresh agent can pick one up without other context.

## Classification legend

- **official-cdn** — `https://wakfu.cdn.ankama.com/gamedata/<version>/*.json` (the goal).
- **official-local-client** — decoded `bdata` tables or `gui.jar` assets shipped in the game client (the goal).
- **semi-official-encyclopedia** — Ankama's own site `www.wakfu.com`, but scraped HTML. Tolerated as a documented exception.
- **non-official** — third-party WakForge / MethodWakfu / Fandom / the `Vertylo/wakassets` PNG mirror.
- **hardcoded-transcribed / hand-maintained** — game-data numbers transcribed from a third-party source or entered by hand, living in source/overlay files.

Status: 🟢 Actionable (a concrete official source exists) · 🟡 Keep / documented exception (unavoidable today, with a research path) · ℹ️ Not a game-data source.
